# Codebase-Init Refinements Design Brief

- **Chosen Approach:** Approach A — Declarative Schema Extension & Rich Conventions Scaffolding (Modified: skipped Code Style/Linter Preference)
- **Why:** Skipping code style/linter preference keeps the initial policy survey focused on high-level workflows. Adding a CI/CD integration rule and replacing the key conventions placeholder with dynamic language-specific defaults directly addresses the YAGNI principle while eliminating empty placeholders.
- **Scope:**
  - **In-scope:**
    - Add a 4th policy question to Phase 0 and `hard-rules.md`: **CI/CD Automation** (`github-actions` / `gitlab-ci` / `local-only`).
    - Update the `codebase-init:hard-rules` marker comment to parse the new `ci` parameter.
    - Expand `scaffold-agents-md` command to dynamically generate language-specific `## Key Conventions` (e.g., specific rules for Node/Python/Go) instead of a static generic TODO block.
    - Update `run.py` linter and unit tests to support the updated marker structure.
  - **Out-of-scope:** Code style/linter preference survey questions.
- **Success Criteria:**
  - `run.py lint-agents-md` passes successfully on the newly generated `AGENTS.md` containing the updated marker.
  - Generated files contain 3-7 actual, language-appropriate key conventions rather than a placeholder checklist.
- **Constraints:** Keep `AGENTS.md` strictly under 100 lines and preserve existing custom sections during regeneration.
- **Interface:**
  - Cli parameter updates: `scripts/run.py scaffold-agents-md` accepts `--ci <github-actions|gitlab-ci|local-only>`.
  - Phase 0 prompt asks 4 questions in a single `AskUserQuestion` call instead of 3.
- **Architecture:**
  - [run.py](file:///C:/agent-dev/skills/codebase-init/scripts/run.py) handles CLI arguments parsing, scaffolding, marker generation, and validation checks.
  - [hard-rules.md](file:///C:/agent-dev/skills/codebase-init/references/hard-rules.md) stores the survey options and recommendations.
- **Risk Register:**
  - _Risk:_ Adding a 4th question makes the initial Phase 0 survey longer. _Mitigation:_ Ensure the recommended choice is pre-selected and accurately derived from the repository's `.github/` or `.gitlab/` directory structure during Phase 1.
- **Review Disposition:** Phase 5 not triggered (Scope M, low blast radius).
- **First Step:** Write the design brief to `docs/design/2026-06-23-codebase-init-refinements-design.md`.
